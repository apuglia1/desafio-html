Entendendo a comunicação Client x Server

Voltar

Nesta aula, iniciamos o aprendizado com o histórico da computação, quais foram as primeiras máquinas e todo o desenvolvimento que se deu até chegamos ao estágio em que nos encontramos. Exemplo disso foram as máquinas como o Mark I, Colossus, dentre outras tantas.

Outro ponto importante foi o fato de que alguns projetos, a exemplo do DARPA, foram fundamentais para o desenvolvimento do embrião do que hoje conhecemos como a Internet; certamente que desse embrião houve a evolução para iniciativas mais ousadas, a exemplo do que fez o Tim Berners-Lee, ao criar o primeiro site da história, em 1991.

Além disso, existem outras figuras importantes, como Vinton Cerf e Robert Kahn, responsáveis pela criação do protocolo TCP/IP.

Outros nomes importantíssimos são:

  • Joseph Carl Licklider, por ter sido o primeiro a ter a ideia de uma rede de computadores ("uma galáxia");
  • Grace Hopper, por ter sido a primeira programadora do Mark I, além de ter sido a inventora do Flow-Matic, que serviu como base do COBOL, que até hoje é usado, especialmente em sistemas bancários;
  • Bill Gates e Steve Jobs também são nomes bastante relevantes e determinantes para entender como a indústria da tecnologia chegou ao patamar atual, tendo em vista as suas contribuições com a Microsoft e a Apple, respectivamente;
  • Uma menção extra que é bastante digna é o nome de Margaret Hamilton, que foi a responsável por escrever manualmente todo o código da nava Apollo 11, responsável por levar o homem à Lua.

O que são clients

Em seguida, foi aprendida a noção de clientes e servidores. Clientes podem ser entendidos como aqueles que solicitam algo a um servidor, podendo ser diversos tipos de dispositivos, como PCs, notebooks, tablets etc., bem como também podem ser um navegador (Mozilla, Vivaldi, Edge, Chrome etc.) ou um aplicativo de email. O conceito de hospedagem, por exemplo, significa dizer que o arquivo com as informações do site ficarão no servidor.

Outra ideia importante foi a de cache, que é uma espécie de "memória" que evita que o navegador precise buscar as informações na Internet toda vez que se acessar determinado site.

Importante também mencionar a necessidade de uma consciência do programador quando desenvolver um site, pois hoje é possível acessar a Internet pelos mais diveros meios - PCs, notebooks, tablets, smartphones -, de tal maneira que uns terão desempenho excelente, outros, nem tanto: portanto, é necessário criar sites que sejam responsivos e que abarquem, sempre que possível, a maior quantidade de dispositivos, proporcionando, assim, uma experiência de qualidade ao usuário.

Estudou-se também as questões relativas às aplicações web. De forma simples, pode-se dizer que depende, se quisermos saber se determinada aplicação é web ou não. Exemplos citados na aula que expressam bem são o YouTube e o LinkedIn; ambos podem ser ou não aplicações web, a depender de como se enxerga. Se estamos falando dos aplicativos que estão disponíveis nos smartphones (podendo ser baixados nas suas lojas de aplicativos), por exemplo, não seria o caso de uma aplicação web; pela via oposta, podemos dizer que, sim, são aplicações web se formos "depender" do uso dessas plataformas por meio de seus sites, usando um navegador e, por consequência, a Internet.

Finalizando o tema clients, foi visto como os dispositivos móveis estão presentes em nossa realidade e como o público que usa dispositivos móveis é fundamental para entender como será o futuro da Internet, tendo em vista as facilidades atreladas a ele, como o seu uso simples, portabilidade e também popularidade.

Sobre servidores

Voltar

O que são servers

Também foi estudado a outra parte, que são os servidores. Estes são a contraparte que oferece algum tipo de serviço ao cliente, podendo ser, por exemplo, um computador dedicado a tal tarefa (servidor web ou banco de dados). É importante mencionar que não é a única alternativa, pois existem inúmeros tipos de redes, a exemplo da LAN (Local Area Network), MAN (Metropolitan Area Network), WAN (Wide Area Network), dentre outras.

Existem servidores para cada tipo de finalidade, desde computadores locais até grandes centros com enormes máquinas, que chamamos de data centers.

Hardwares e Softwares em Servidores

Não há diferença entre o conceito de hardware para servidor ou um computador para uso pessoal; são as peças que compõem a parte física da máquina. Contudo, a se julgar pelo número de operações que o servidor realizará, o que se poderia dizer é que se espera um considerável poder de processamento e por isso mesmo, uma maior robustez de peças.

Já no que diz respeito ao software, há determinadas peculiaridades, onde o sistema operacional, por exemplo, possuirá recursos voltados para servidores. Há, aqui, uma força considerável do sistema Linux com suas diversas distribuições, a exemplo do Debian, CentOS, Ubuntu e Fedora. Mas, por outro lado, também há o Windows Server e outras alternativas, como o Oracle. O Linux é considerado um grande SO "server sided", dentre outros motivos, em razão de sua segurança.

No que diz respeito aos Servidores Web, existem servidores conhecidos, como é o caso do Apache e do Nginx ou mesmo o LiteSpeed Web Server ou Microsoft IIS.

A requisição HTTP é um pedido de informação: de forma análoga, é como se o cliente pedisse algo. O sistema operacional analisa e determina que o servidor execute a requisição, fornecendo as informações necessárias.

Tipos de servidores

Existem muitas variedades de servidores. Alguns exemplos são: a) Servidor proxy, que pode servir como uma espécie de "censor" e que antes de ser executada a requisição na Internet, faz algumas checagens e pode, por exemplo, bloquear certos sites - em resumo, serve como intermediário entre o cliente e o servidor; b) Servidor firewall também é voltado para a segurança, mas neste caso para acessos externos. Por exemplo, evitaria que arquivos maliciosos sejam enviados a um dos clientes que estejam acessando a Internet; c) E-mail server, que é o local para armazenar os e-mails; d) Database server, que é um servidor voltado para o banco de dados; e) DNS (Domain Name Service), que traduz o nome dos sites, convertendo-o para o IP, que é como se fosse o CPF do site, uma identidade única.

Hospedagem de sites

Hoje em dia, muitas empresas oferecem pacotes bastante completos para a pessoa que deseja criar o seu site, já fornecendo planos em que é garantida a hospedagem, o domínio, além de certos benefícios. Uma analogia interessante para se entender o domínio, por exemplo, seria a de compará-lo com a compra de um terreno onde se construirá um imóvel; ele é o endereço e também a própria construção.

Linguagens de programação

Voltar

O que é

Linguagens de programação são códigos utilizados para a comunicação entre o homem e computador. Considerando que o computador apenas entende a linguagem binária (0 e 1), ao longo da história foram criadas formas de melhor comunicação entre homem e máquina, o que possibilitou o avanço e a criação de novas tecnologias nesse sentido.

As linguagens de programação podem ser consideradas de baixo nível ou de alto nível. As de baixo nível podem ser entendidas como sendo mais próximas da linguagem de máquina do que da linguagem humana, de maneira que são consideravelmente mais complexas. Exemplo de linguagem de baixo nível seria o Assembly cuja disposição de código dificilmente seria compreendida por um humano, a menos que fosse treinado para compreendê-la. Por sua vez, linguagens de alto nível são aquelas mais próximas da linguagem humana e que por isso mesmo possibilitam um melhor entendimento, sem prejuízo da interação com a máquina. Exemplos são Python e Java.

Considerando as linguagens populares, não existe um consenso, nem há de fato uma linguagem superior a outra. Por exemplo, Python é uma excelente linguagem para análise de dados, enquanto Java é muito utilizado em sistemas legados; Golang, por sua vez, é talvez uma das poucas linguagens que permite que os aplicativos sejam executados em vários núcleos ao mesmo tempo; outras linguagens são voltadas para desenvolvimento mobile etc.

Server-side

As linguagens server-side podem ser entendidas como linguagens back-end, a exemplo do PHP, Java, Ruby, C# etc. Além disso, existem outros exemplos, como o Golang, Python, dentre outras.

Pode ocorrer de determinadas linguagens serem mais adequadas para determinados projetos, mas em regra, não existe, como já mencionado, uma linguagem que seja superior ou melhor. Isso vai muito da experiência do programador, seu background, e de quais linguagens ele se familiarizou durante esse trajeto. Por isso mesmo, é perfeitamente possível usar C#, Python ou Java no back-end.

Client-side

No caso do client-side, costuma-se dizer que é o front-end, isto é, tudo aquilo que se vê no site, a parte estética. Para que haja essa parte estética, geralmente se faz uso da famosa tríade HTML - CSS - JavaScript. O HTML faz o markdown, isto é, determina o posicionamento dos elementos, e gera a estrutura do site; o CSS estiliza e dá mais vida aos elementos do HTML, além de dar algum dinamismo; por fim, o JavaScript é a linguagem de programação que é utilizada, que permite uma série de funcionalidades novas. Javascript é uma linguagem de programação extremamente popular, podendo rodar tanto no back-end (server-side), por meio do Node.js (que no fim das contas, quem roda é o V8), e também no front-end (client-side), este, pelo navegador.

HTML não é uma linguagem de programação

O título nesse caso é autoexplicativo. HTML não é uma linguagem de programação, mas, como o próprio nome diz, uma linguagem de marcação (HyperText Markdown Language). Isso significa que o HTML serve para determinar a posição de elementos no site, tais como os locais de parágrafos, títulos, ancoragem, links, formulários, cabeçalho, rodapé etc, permitindo que o navegador consiga interpretá-los e exibi-los. Neste caso, a linguagem de programação, das três citadas, seria o JavaScript.



HTML na prática

Voltar

Inspetor de elementos

O inspetor de elementos é uma ferramenta muito útil para todo tipo de desenvolvedor web, seja iniciante, intermediário ou avançado. Em resumo, é uma ferramenta fornecida pelos navegadores que possibilita ter uma noção exata dos elementos da página, ter uma ideia do tamanho da margem ocupada por determinado parágrafo, por exemplo.

Estrutura básica do HTML

O HTML possui uma estrutura básica que precisa ser respeitada para que seja aproveitada ao máximo e, mais importante, para que funcione. Ele é case insensitive, isto é, não há mudança de resultado ou erro caso seja digitado em caixa alta ou não. A estrutura precisa estar entre "<>".Algumas de suas partes mais importantes são:

  • HTML: é o início. Tudo que estiver dentro da tag HTML será lida e interpretada pelo navegador de alguma forma.
  • Head: é nesta tag onde fica, por exemplo, a tag title, que permite a mudança do nome da página, no topo do navegador.
  • Body: é o corpo do HTML. É neste trecho em que se insere os títulos, parágrafos, links, imagens, vídeos, formulários etc.
  • Footer: de forma simplificada, seria o rodapé do site.

Falando sobre tags

Existem inúmeras tags no HTML. Para a formatação de texto, por exemplo, existem os mesmos tipos que existem em editores de texto, a exemplo do negrito (aqui representado pela tag "strong"), itálico (representado pela tag "i"), a exemplo da strong, que permite que o texto fique em negrito; a tag i, que permite que o texto fique em itálico; a tag

Atributos das tags

Atributos possuem diversas funções. Algumas dessas funções são:

  1. Style: é possível estilizar um determinado trecho do HTML. No entanto, não é recomendável utilizar esse atributo dentro do HTML, sendo de boa prática utilizar o CSS para tanto.
  2. Src: é o mesmo que "source". É possível utilizando a tag img, fazer uso de uma imagem extraída da internet e colocá-la no site, por exemplo.
  3. Id: é um identificador para apenas um elemento HTML, devendo ser único em toda a página.
  4. Class: é um identificador que serve para agrupar elementos HTML. É fundamental para aplicar estilos a vários grupos no CSS (mesmas cores, mesmo tamanho, mesma disposição etc.).

Textos

Para criar textos no HTML, é importante conhecer a base, que neste caso são os headers e o parágrafo.

Os headers são os títulos e são divididos entre 1 (cujo tamanho é o maior) até o 6 (menor tamanho). O header é um exemplo de tag que precisa ser aberta e fechada, sempre lembrando de colocar o número relativo ao tamanho que se busca; por exemplo, se pretende um header de tamanho 5, basta usar "h5"" e "/h5" (com os sinais de <> e , respectivamente), inserindo o título que se pretende entre as tags.

Por sua vez, os parágrafos complementam o título, constituindo uma boa parcela do que se irá incluir no site. Para fazer uso do parágrafo, basta usar a tag "p". Também é uma tag que precisa de abertura e fechamento.

Listas ordenadas e não ordenadas

Existe a possibilidade criar algumas espécies de listas quando se utiliza o HTML. Essas listas podem ser ordenadas, isto é, itens numerados, ou listas não ordenadas, que como não possuem uma ordem, geralmente são antecedidas por círculos (embora isso possa ser alterado no CSS para outras formas, como quadrados, ou mesmo, retirá-los).

Para fazer uso da lista ordenada, usamos inicialmente a tag "ol" (que significa "ordered list") e, dentro da tag, inserimos uma tag filha, que no caso é a "li" para cada item que queremos inserir. O resultado será como visto abaixo:

Links

Para entender o que é link, é necessário entender o que é âncora. Em HTML, ela é representada pela tag "a". Ancorar pode ser entendido também "fixar", "prender", da mesma forma que uma âncora de navio funciona.

Em todo caso, para se criar uma página contendo um link, basta que se crie a tag a, que irá abrir o href="" - neste local entre aspas é que se coloca o link que se quer acessar. A tag "a" também precisa ser aberta e fechada, de maneira que, entre a abertura e fechamento, é possível escrever algo, que muito provavelmente conterá alguma informação a respeito do link que será acessado.

Vale ressaltar que ao se acessar o link, o navegador irá, por padrão, abrir na mesma página. Essa definição padrão é chamada de "_self"; para que o navegador abra a página em outra aba, basta incluir o atributo target=_blank dentro da tag "a".
